# 新闻主题与板块匹配优化方案

## 1. 背景与目标

当前系统的新闻舆情、风险拆解、龙头战法、板块归因已经具备基础能力，但仍存在三个核心问题：

- 个股新闻主要依赖股票名称匹配，漏掉大量不带股票名但明显影响板块的新闻。
- 板块盘面新闻容易被误判为个股利好/利空，例如“PCB概念拉升，合力泰跟涨”不应判为合力泰利好。
- 个股所属板块仍偏依赖静态概念、涨停原因和同花顺板块关系，缺少“新闻显式点名板块和个股动作”的动态证据。

本方案目标是建立一套统一的“新闻主题识别 + 股票关系抽取 + 板块匹配评分”机制，让新闻舆情、风险拆解、龙头战法和板块归类使用同一套底层判断。

核心原则：

- 新闻舆情只判断可归因到个股的基本面/公告/监管事件。
- 板块盘面新闻用于板块归因和市场热度，不直接判个股利好利空。
- 个股所属主线题材由多源证据打分决定，而不是单一字段命中。

## 2. 当前问题拆解

### 2.1 个股新闻名称匹配覆盖不足

现有新闻查询主要使用股票名称在标题和正文中模糊匹配：

```text
title LIKE %stock_name% OR content LIKE %stock_name%
```

这种方式会漏掉以下新闻：

- 只写公司全称，不写股票简称。
- 只写股票简称别名、曾用名。
- 只写业务/产品/项目，不写股票名。
- 只写板块新闻，例如“算力租赁板块走强，多股涨停”。
- 新闻里明确列了相关股票，但当前股票不在标题中。

### 2.2 板块盘面新闻与个股情感混淆

例如：

```text
PCB概念股盘初拉升，博敏电子涨停，广合科技、合力泰、国际复材、强瑞技术跟涨。
```

正确结论应该是：

```text
合力泰板块归因：PCB
合力泰在该新闻中的角色：跟随股
个股新闻情感：中性
板块热度：正向
```

错误结论是：

```text
合力泰利好
```

系统必须区分“个股基本面利好”和“板块行情跟随”。

### 2.3 涨停原因不能直接等于标准概念

`lu_desc` 通常是自然语言组合：

```text
半导体洁净室+海外EPC+城市更新
AI算力+数据中心+云服务
铜缆高速连接+PCB
```

这些词需要映射到标准概念：

```text
半导体洁净室 -> 芯片概念 / 半导体
AI算力 -> 算力租赁 / 人工智能 / 数据中心
铜缆高速连接 -> 铜缆高速连接 / CPO
海外EPC -> 一带一路 / 建筑装饰
```

没有标准化会导致板块匹配分散，或者误匹配到泛概念。

### 2.4 静态概念容易把主线带偏

同花顺 `ths_member` 中的静态概念数量较多，其中很多是泛概念：

```text
一带一路、华为概念、国企改革、人工智能、数字经济
```

这些概念存在覆盖面广、解释力弱的问题。如果没有当日新闻、涨停原因或热点榜支撑，不应优先作为主跟随题材。

### 2.5 多主题冲突缺少统一评分

一只股票可能同时命中多个主题：

```text
合力泰：PCB、算力租赁、消费电子
金螳螂：商业航天、芯片概念、一带一路
中嘉博创：算力租赁、人工智能、数据中心
```

当前容易出现“命中哪个先显示哪个”的问题。应改为多证据打分，选出当天最强主线。

## 3. 总体架构设计

新增三层能力：

```text
NewsThemeExtractor
  从新闻中抽取主题、动作、股票、角色、强度

ThemeAliasResolver
  统一概念别名、涨停原因词、新闻主题词

ThemeAttributionScorer
  根据多源证据给股票-主题关系打分，选出主跟随题材
```

统一数据流：

```text
新闻库 news_data
  -> NewsThemeExtractor
  -> news_theme_relation
  -> ThemeAttributionScorer
  -> 个股主跟随题材 / 板块热度 / 风险拆解依据
```

## 4. 优化项 1：新闻主题关系抽取

从新闻标题和正文中抽取结构化关系：

```text
板块/概念名：PCB、算力租赁、商业航天、半导体
触发动作：拉升、走强、爆发、回落、走低
个股角色：leader / follow / drag / mentioned
个股动作：涨停、跟涨、涨超10%、跌超5%、回落
时间短语：盘初、早盘、午后、尾盘、收盘、昨日
强度：涨停、涨超10%、多股联动、ETF上涨、成交放量
```

示例：

```text
新闻：PCB概念股盘初拉升，博敏电子涨停，广合科技、合力泰、国际复材跟涨

抽取：
theme = PCB
stock = 合力泰
role = follow
action = 跟涨
time_phrase = 盘初
theme_sentiment = positive
stock_sentiment = neutral
```

当前已完成原型：

- `backend/services/news_theme_extractor.py`
- `scripts/test_news_theme_extraction.py`
- `data/news_theme_extraction_sample.json`

## 5. 优化项 2：句子级绑定

长新闻里可能同时出现多个板块和多组股票：

```text
算力租赁方向延续强势，中嘉博创7天4板，三人行5天3板。
商业航天方向走高，金螳螂12天10板，宇环数控4天3板。
绿电方向活跃，华电辽能10天6板。
```

不能把所有股票都绑定到所有板块。

处理方式：

- 先按 `。；;！\n` 切分句子。
- 每个句子独立抽取主题和股票。
- 只在同一句或相邻短句内建立主题-股票关系。
- 对“其中、此外、另一方面、板块方面”做段落上下文继承。

目标：

```text
金螳螂 -> 商业航天
中嘉博创 -> 算力租赁
华电辽能 -> 绿电
```

避免：

```text
金螳螂 -> 算力租赁
中嘉博创 -> 商业航天
```

## 6. 优化项 3：全市场股票映射

当前原型只用 `selected_stock` 中的股票名，覆盖面不足。正式版应使用全市场股票基础表。

数据来源优先级：

- Tushare `stock_basic`
- 本地已有股票基础表
- 历史选股结果补充

需要字段：

```text
ts_code
symbol
name
fullname
market
exchange
list_status
industry
```

别名规则：

```text
股票简称：合力泰
公司全称：合力泰科技股份有限公司
代码：002217 / 002217.SZ
去空格简称：金螳螂 <- 金 螳 螂
```

同时要建立排除词：

```text
截至收盘、创业板指、能源金属、半导体、ETF、指数、成交额
```

这些不能被识别成股票。

## 7. 优化项 4：涨停原因标准化映射

建立 `ThemeAliasResolver`，把 `lu_desc` 和新闻主题统一到标准概念。

建议表：

```text
theme_alias_map
- id
- alias_name
- normalized_theme_name
- source
- weight
- is_active
- created_at
- updated_at
```

示例规则：

```text
半导体洁净室 -> 半导体 / 芯片概念
AI算力 -> 算力租赁 / 人工智能
智算中心 -> 算力租赁 / 数据中心
铜缆高速连接 -> 铜缆高速连接 / CPO
PCB -> PCB概念
印制电路板 -> PCB概念
商业航天 -> 商业航天 / 卫星互联网
海外EPC -> 一带一路
```

注意：

- 标准化不等于唯一映射，一个别名可映射多个候选主题。
- 泛概念需要降权，比如一带一路、国企改革、华为概念。
- 当 `lu_desc` 与新闻主题一致时，权重大幅提高。

## 8. 优化项 5：主题归因打分器

新增 `ThemeAttributionScorer`，对每个股票-主题候选打分。

建议评分：

```text
涨停原因命中主题：+35
新闻标题中明确“主题 + 股票动作”：+30
新闻正文中同句绑定：+22
股票角色为 leader：+15
股票角色为 follow：+10
主题进入热点榜前5：+20
主题进入热点榜前10：+12
板块涨停家数 >= 5：+10
同花顺 ths_member 静态归属：+8
selected_stock.concept 命中：+5
行业兜底命中：+3
泛概念降权：-8
新闻超过3个交易日：按时间衰减
```

输出：

```text
primary_theme
theme_score
evidence_list
candidate_themes
confidence
```

## 9. 优化项 6：降级链路重构

推荐正式板块匹配链路：

```text
1. lu_desc 标准化主题
2. 新闻主题显式点名该股
3. limit_cpt_list 热点榜交叉
4. ths_member 个股概念
5. selected_stock.concept / industry
6. 行业兜底
```

当前系统已经有多源降级，但新闻主题关系尚未正式接入。接入后，新闻主题应位于高优先级，因为它反映“当天市场怎么讲这只股票”。

## 10. 优化项 7：新闻情感与板块归因解耦

明确两类结论：

```text
个股新闻情感：positive / negative / neutral
板块主题情绪：positive / negative / neutral
```

示例：

```text
新闻：PCB概念拉升，合力泰跟涨

个股新闻情感：neutral
板块主题情绪：positive
个股板块归因：PCB
```

只有以下新闻可判断个股利好/利空：

- 业绩公告
- 减持/增持/回购
- 监管函/立案调查/处罚
- 中标/重大合同
- 重大资产重组
- 退市/ST风险
- 公司明确公告或投资者关系记录

板块盘面新闻、新闻精选、行情回顾默认不做个股情感。

## 11. 优化项 8：新闻去噪和可信度分层

新闻不是都一样可信，应分层处理。

高可信：

- 个股公告
- 投资者关系记录
- 龙虎榜/交易所公开信息
- 新闻标题明确点名股票和事件

中可信：

- 板块新闻中同句点名股票
- 行业新闻明确涉及公司主营业务

低可信：

- ETF营销稿
- 市场收评
- 新闻精选合集
- 海外市场新闻
- 大盘指数新闻

处理规则：

- 高可信可进入风险拆解和情感评分。
- 中可信进入板块归因和热度分析。
- 低可信只作为背景，不直接影响个股结论。

## 12. 优化项 9：时间衰减

板块归因必须考虑时间。

建议衰减：

```text
当日盘中/盘后：100%
昨日复盘：70%
2-3个交易日前：40%
超过5个交易日：仅保留历史背景，不参与主线判断
```

同一股票的主线题材应优先使用最近交易日新闻。

## 13. 优化项 10：多主题冲突处理

一只股票多个主题同时命中时，按分数排序。

示例：

```text
合力泰：
PCB：新闻点名跟涨 + 标题主题 + 当日热点 = 72
算力租赁：旧新闻点名 + 非当日 = 45
消费电子：静态概念 = 12

主跟随题材：PCB
候选题材：算力租赁、消费电子
```

前端展示不能只显示一个结论，应显示候选主题和证据：

```text
主跟随题材：PCB（72分）
证据：
- 新闻《PCB概念股盘初拉升，博敏电子涨停》点名合力泰跟涨
- PCB主题当日盘初拉升
- 角色：跟随股

候选题材：
- 算力租赁（45分）
- 消费电子（12分）
```

## 14. 优化项 11：证据可解释展示

风险拆解和龙头战法应避免只显示短句，例如：

```text
所属题材排名第1，市场主线题材
```

应改成：

```text
主跟随题材：PCB
依据：
1. 新闻《PCB概念股盘初拉升，博敏电子涨停》点名合力泰跟涨
2. 该新闻主题动作为“盘初拉升”，板块情绪正向
3. 股票角色为 follow，不判定为个股利好，只作为板块归因
```

这能避免用户误解“新闻利好”和“板块热度”。

## 15. 优化项 12：新闻主题关系入库

建议新增表：

```text
news_theme_relation
- id
- news_id
- trade_date
- publish_time
- source
- title
- theme_name
- normalized_theme_name
- ts_code
- stock_name
- role
- action
- action_strength
- time_phrase
- sentiment_for_theme
- confidence
- evidence
- created_at
- updated_at
```

用途：

- 选股后快速查某股新闻主题关系。
- 板块归因无需每次重新扫新闻文本。
- 支持回溯和人工校验。

## 16. 优化项 13：人工校验集与回归测试

需要建立小样本人工标注：

```text
002217 合力泰 -> PCB / follow / 跟涨
000889 中嘉博创 -> 算力租赁 / mentioned 或 follow
002081 金螳螂 -> 商业航天 / leader
001316 润贝航科 -> 商业航天 / leader
```

每条标注包含：

```text
ts_code
stock_name
expected_theme
expected_role
expected_action
source_news_title
should_affect_sentiment: true/false
```

回归测试目标：

- 板块新闻不误判为个股利好。
- 复盘长新闻不跨句串主题。
- 涨停原因能映射到标准概念。
- 泛概念不会压过当日强主题。

## 17. 实施阶段

### 阶段一：高置信新闻主题抽取

已完成原型：

- `NewsThemeExtractor`
- 新闻库样本抽取脚本
- 合力泰 PCB 关系测试

下一步：

- 接入全市场股票基础表。
- 加入句子级绑定。
- 过滤 ETF 营销稿和纯指数新闻。

### 阶段二：涨停原因标准化

新增：

- `ThemeAliasResolver`
- `theme_alias_map`
- 高频热点别名规则

目标：

- `lu_desc` 可以稳定映射到标准概念。
- 新闻主题和涨停原因使用同一套标准名。

### 阶段三：主题归因评分

新增：

- `ThemeAttributionScorer`
- `stock_theme_attribution` 缓存表

目标：

- 输出主跟随题材、候选题材、证据、置信度。
- 接入风险拆解和龙头战法。

### 阶段四：前端解释增强

风险拆解展示：

- 主跟随题材
- 候选题材
- 新闻证据
- 涨停原因证据
- 热点榜证据
- 是否为“板块归因，不构成个股利好”

```text

```

期望：

```text
板块归因：PCB
角色：follow
动作：跟涨
个股情感：neutral
```

### 18.2 中嘉博创样例

新闻：

```text
AI应用、算力租赁方向延续强势，中嘉博创7天4板
```

期望：

```text
板块归因：算力租赁
角色：mentioned 或 leader（根据句子级规则）
个股情感：neutral
```

### 18.3 金螳螂样例

新闻：

```text
核心高标金螳螂12天10板，带动商业航天方向走高
```

期望：

```text
板块归因：商业航天
角色：leader
个股情感：neutral
```

### 18.4 涨停原因样例

涨停原因：

```text
半导体洁净室+海外EPC+城市更新
```

期望候选：

```text
半导体 / 芯片概念
一带一路
城市更新
```

但如果当日新闻或热点榜支持半导体，则主跟随题材优先半导体。

## 19. 结论

这套优化的关键不是“多抓新闻”，而是把新闻拆成两类价值：

- 个股事件价值：用于新闻舆情、风险拆解。
- 板块归因价值：用于判断股票当天跟随哪个市场主题。

最终系统应该做到：

```text
同一条新闻：
既不会误判成个股利好；
又能正确提供板块归因证据。
```

这样风险拆解、龙头战法、新闻舆情和板块匹配才能保持同一套逻辑口径。
