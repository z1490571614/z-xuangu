## 1. 模块概述

### 1.1 定位

本模块是系统的**底层公共情感判定引擎**，负责将新闻/公告文本转化为结构化、可解释、可复用的情感判定结果。它是一个**纯代码规则模块**，不调用任何 AI 大模型，不绑定任何具体策略。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **事件驱动** | 先识别事件类型，再在事件内部评分，而非简单关键词利好/利空匹配 |
| **纯规则** | 只使用关键词规则、正则表达式、事件优先级、数值抽取、条件判断 |
| **不绑定策略** | 只输出基础情感结果，不输出"买入/卖出"等交易建议 |
| **可解释** | 每条结果包含匹配规则链、风险标记、理由说明 |
| **确定性分离** | 情感方向与确定性因子分离，不确定性词不直接判中性 |

### 1.3 被调用方

| 调用方 | 模块路径 | 用途 |
|--------|----------|------|
| 龙头战法评分 | `dragon_leader/scorer/announcement_alpha.py` | 消息面评分，映射到财务/股东/监管维度 |
| 风险拆解 | `dragon_leader/scorer/retreat_scorer.py` | 从 dimension_scores 读取风险维度数据 |
| 新闻舆情API | `api/news_v2.py` | 返回新闻时实时分析情感 |
| 个股详情API | `api/stock_detail.py` | /news 端点情感分析 |
| 龙头战法main | `dragon_leader/main.py` | collect_news 中重新分析情感 |

## 2. 核心功能说明

### 2.1 功能清单

| 功能 | 说明 |
|------|------|
| 文本标准化 | 清洗空格、换行、全角符号，拼接标题与正文 |
| 新闻范围分类 | 区分单股新闻、多股综述、市场综述 |
| 事件类型识别 | 扫描19种事件类型候选，按优先级选择主事件 |

| 多事件冲突合并 | 同条新闻多个事件时，按冲突规则合成最终分 |

## 4. 算法原理与实现细节

### 4.1 文本标准化

**位置**: `normalizer.py`

```
输入: title + content
处理: ① 拼接 ② 去除空格/换行/制表符 ③ 全角% → 半角% ④ 全角逗号句号 → 半角
输出: 连续无空格文本
```

**数值抽取工具函数**:
- `extract_percentage()`: 正则提取 `(\d+\.?\d*)%`，返回 float 列表
- `extract_amount()`: 提取金额，万亿→1e12，亿→1e8，万→1e4，元→1，返回 元 单位
- `extract_stock_count()`: 提取股数，万亿股→1e12，亿股→1e8，万股→1e4

### 4.2 载体词处理

**定义**:
```python
CARRIER_WORDS = ["公告", "公告称", "发布公告", "披露", "发布", "刊发", "刊登", "表示", "称", "显示"]
```

**规则**: 载体词**不参与情感判断**。它们只是信息发布载体，不表达任何利好或利空。

**示例**:
- 输入: `"公司公告称，2024年净利润同比增长1309.44%"`
- 处理: 忽略"公告称"→ 识别为 performance 事件 → 净利润同比增长1309.44% → 强正面

### 4.3 不确定性词处理

**定义**:
```python
UNCERTAINTY_WORDS = ["拟", "计划", "筹划", "预计", "有望", "或将", "框架协议", ...]
```

**规则**: 不确定性词**只影响 certainty_factor**，不直接决定情感方向。

**示例**:
- `"预计净利润同比增长1300%"` → sentiment=positive, certainty=forecast, factor=0.85
- `"拟减持"` → sentiment=negative, certainty=planned, factor=0.70

**确定性类型与因子**:

| 类型 | 系数 | 说明 |
|------|------|------|
| completed | 1.00 | 已完成 |
| signed | 0.95 | 已签署 |
| approved | 0.90 | 已审议通过 |
| forecast | 0.85 | 预计 |
| planned | 0.70 | 拟/计划 |
| preliminary | 0.55 | 筹划/初步意向 |
| framework | 0.45 | 框架协议 |
| uncertain | 0.35 | 存在不确定性 |
| unknown | 1.00 | 未知 |

### 4.4 情感分类算法

**位置**: `constants.py` — `score_to_sentiment()`

```python
def score_to_sentiment(score, positive_score=0, negative_score=0):
    # 混合检测
    if positive_score >= 1.5 and negative_score <= -1.5:
        if abs(negative_score) >= 3.0:
            return "negative"  # 重大负面覆盖
        return "mixed"         # 多空混合
    # 单方向判定（阈值 ±0.2）
    if score > 0.2:  return "positive"
    if score < -0.2: return "negative"
    return "neutral"
```

**关键特性**:
- 情感方向与影响强度**分离判定**
- 阈值 `±0.2`（而非旧版的 `±0.8`），确保小幅负面不被 neutral 吞掉
- 混合检测：同时存在正负事件时，重大负面（≤ -3.0）强制判 negative

### 4.5 影响等级映射

**位置**: `constants.py` — `score_to_impact_level()`

| 绝对值范围 | 等级 | 说明 |
|-----------|------|------|
| ≥ 4.0 | critical | 极强 |
| ≥ 3.0 | strong | 强 |
| ≥ 1.5 | medium | 中等 |
| ≥ 0.3 | weak | 弱 |
| < 0.3 | none | 无 |

### 4.6 置信度计算

**位置**: `confidence.py`

```python
confidence = 0.5                    # 基础置信度
if event_type != "other":           +0.2  # 事件类型明确
if facts 有值:                      +0.15 # 抽取到事实
if source 是公司/交易所公告:        +0.1  # 来源权威
if content 长度 < 20:               -0.15 # 正文太短
```

范围: 0.0 ~ 1.0

### 4.7 多事件冲突合并

**位置**: `scorer.py` — `merge_event_scores()`

```python
positive_score = sum(正分数)
negative_score = sum(负分数)
min_negative = 最小负分数

if min_negative <= -3.0 and positive_score > 0:
    # 重大负面优先：负分全额 + 正分仅20%抵消
    final_score = min_negative + positive_score * 0.2
    risk_flags.append("positive_event_overridden_by_major_risk")
else:
    # 普通多空：直接相加
    final_score = positive_score + negative_score
```

## 5. 数据处理流程

### 5.1 单条新闻分析完整流程

```
analyze_news_event(news_item, context=None, debug=False)
  │
  ├─ 1. normalize_text(title, content)
  │     输出: 清洗后的连续文本
  │
  ├─ 2. classify_news_scope(title, content, stock_name)
  │     输出: {news_scope, reason, has_target_stock}
  │
  ├─ [分支A] 如果是 multi_stock / market_overview
  │     └─ _handle_multi_stock()
  │          ├─ extract_target_context(text, stock_name)
  │          ├─ 未传入/未找到 → unrelated, score=0
  │          └─ 找到 → _score_local_context() → 局部弱评分
  │
  ├─ [分支B] 如果是 single_stock / unknown（标准流程）
  │     ├─ 3. classify_event_candidates(text)
  │     │     扫描19个事件类型的关键词，返回匹配列表
  │     ├─ 4. select_primary_event(candidates)
  │     │     按优先级选主事件
  │     ├─ 5. _extract_facts(event_type, text, context)
  │     │     按事件类型调用对应抽取函数
  │     ├─ 6. score_event(event_type, text, facts)
  │     │     分发到 rules/ 各模块 → (raw_score, subtype, risk_flags, certainty)
  │     ├─ 7. 对所有候选事件评分（用于冲突合并）
  │     ├─ 8. merge_event_scores(all_scores)
  │     │     重大负面优先合并 → merged_score
  │     ├─ 9. 确定性识别 + certainty_factor
  │     ├─ 10. final_score = merged_score × certainty_factor
  │     ├─ 11. score_to_sentiment() + score_to_impact_level()
  │     └─ 12. _build_result() → 结构化输出
  │
  └─ [debug=True] 附加 debug_info
```

### 5.2 新闻范围分类逻辑

**位置**: `news_scope.py` — `classify_news_scope()`

```python
# 判断依据:
market_hits = 市场综述词命中数 (竞价看龙头/市场情绪/板块/涨停/连板等)
single_hits = 单股公告词命中数 (公告称/净利润/减持/中标等)
has_many_stock_like_phrases = 多股结构特征 (≥3个"（"或"板"或 高开+低开+涨停)

判定规则:
  ① market_hits ≥ 2 且 has_many_stock_like_phrases → multi_stock
  ② has_target_stock 且 single_hits ≥ 1 → single_stock
  ③ market_hits ≥ 2 → market_overview
  ④ 否则 → unknown
```

### 5.3 多股新闻的局部评分

**位置**: `analyzer.py` — `_score_local_context()`

当目标股票出现在多股新闻中时，只提取目标股票附近 ±80 字符的上下文做局部评分：

| 关键词 | 分值 | 说明 |
|--------|------|------|
| 高开 | +0.3 | 竞价高开 |
| 连板/板 | +0.3 | 连板表现 |
| 业绩超预期/超预期 | +0.6 | 业绩催化 |
| 涨停 | +0.3 | 涨停 |
| 低开 | -0.3 | 竞价低开 |
| 跌停 | -0.5 | 跌停 |

**限制**: 局部评分范围 `[-1.5, +1.5]`，不超过弱到中等强度。

### 5.4 数据流向：入库 → 查询 → 情感分析

```
[采集阶段]                              [查询阶段]

  │                                        │
  ▼                                        ▼
news_data 表 (原始数据) ──→ get_stock_news()
  │                                        │

  │                                   analyze_news_event()
  │                                        │
```

## 6. 事件类型与评分规则

### 6.1 业绩类 (performance)

**识别关键词**: 净利润、营业收入、同比增长、同比下降、扭亏为盈、预增、预亏、续亏、减亏、亏损扩大、不及预期、超预期

**评分原则**:
- **同比优先**：净利润/营收同比变化是主要评分依据
- **环比只做修正**：环比扭亏不能覆盖同比下降（仅 +0.2 弱修正）
- **营收+净利润双降**：强制判 negative（score=-0.3最低）

**评分分段**:

| 净利润同比变化 | 得分 |
|---------------|------|
| ≥ 300% | +4.0 |
| ≥ 100% | +3.0 |
| ≥ 50% | +2.0 |
| ≥ 20% | +1.0 |
| < 0（小幅下降） | -0.6 |
| ≤ -20% | -1.0 |
| ≤ -50% | -2.0 |
| ≤ -80% | -3.0 |

**附加语义**:

| 语义 | 得分 |
|------|------|
| 扭亏为盈 | +2.5 |
| 亏损扩大 | -2.5 |
| 亏损收窄 | +0.8 |
| 非经常性损益 | -0.8 |
| 超预期 | +1.0 |
| 不及预期 | -1.5 |

**风险标记**:
- `missing_deducted_profit_yoy`: 有净利润同比但无扣非净利润同比
- `non_recurring_gain_risk`: 文本提到"非经常性损益"

### 6.2 重组并购 (restructure)

| 状态 | 得分 | 说明 |
|------|------|------|
| 终止重组/重组失败 | -3.5 | 强利空 |
| 不构成重大调整 | 0.0 | 中性 |
| 获得核准/通过审核 | +3.0 | 强利好 |
| 签署正式协议 | +2.0 | 中等利好 |
| 筹划重组 | +1.2 | 弱利好（低确定性） |
| 拟收购 | +1.0 | 弱利好（低确定性） |
| 一般提及 | +0.5 | 极弱 |

### 6.3 减持 (reduce_holding)

| 维度 | 扣分规则 |
|------|---------|
| 持有人类型 | 控股股东/实控人 -2.5, 董监高 -1.5, 其他 -1.0 |
| 减持比例 | ≥5% 额外 -2.5, ≥2% -1.5, ≥1% -0.8 |
| 清仓式 | 额外 -2.0 |
| 被动减持 | 总分 ×0.5（减轻） |
| 完成实施 | +0.5（利空出尽） |

### 6.4 增持 (increase_holding)

| 维度 | 加分规则 |
|------|---------|
| 持有人类型 | 控股股东/董事长 +1.5, 董监高 +1.0, 其他 +0.5 |
| 增持金额 | ≥1亿 额外 +1.5, ≥3000万 +1.0, ≥500万 +0.3 |

### 6.5 回购 (buyback)

| 回购金额/市值比 | 得分 |
|----------------|------|
| ≥ 3% | +2.5 |
| ≥ 1% | +1.5 |
| ≥ 0.3% | +0.5 |
| 无比例 | +0.5（基础分） |

### 6.6 订单合同 (order_contract)

| 合同金额/去年营收比 | 得分 |
|-------------------|------|
| ≥ 50% | +3.5 |
| ≥ 20% | +2.5 |
| ≥ 10% | +1.5 |
| ≥ 3% | +0.5 |

### 6.7 监管处罚 (regulatory)

| 严重程度 | 得分 |
|---------|------|
| 强制退市/退市风险/财务造假 | -5.0 |
| 立案调查 | -4.0 |
| 行政处罚 | -3.0 |
| 监管措施 | -2.0 |

**监管类优先级最高**（EVENT_PRIORITY = 1），不会被普通利好覆盖。

### 6.8 其他事件类型

详见各规则模块源码：
- 问询关注 (inquiry): -1.5 ~ +0.2
- 诉讼仲裁 (litigation): -3.0 ~ +1.0
- 解禁 (unlock): -3.0 ~ -0.3（视解禁市值/流通市值比）
- 股权质押 (pledge): -1.5 ~ +0.8

- 流程公告 (process): 0.0（严格中性）

## 7. 新闻范围分类与多股处理

### 7.1 为什么需要范围分类

旧版情感分析将所有财经文本都当成"单股新闻"，导致以下误判：

| 新闻类型 | 旧版误判 | 正确判定 |
|----------|---------|---------|
| 竞价看龙头(多股) | 判为利好 | 市场综述，不能归因给单股 |
| 人气板块点评(多股) | 判为利好 | 局部上下文判断 |
| 市场综述含回暖词 | 判为利好 | 市场情绪，不归因给单股 |

### 7.2 三种新闻范围的处理策略

| 范围类型 | 未传入股票 | 传入股票但未出现 | 传入股票且出现 |
|----------|-----------|----------------|---------------|
| single_stock | - | 正常事件分析 | 正常事件分析 |
| multi_stock | neutral, score=0 | unknown/unrelated, score=0 | 局部上下文评分 |
| market_overview | neutral, score=0 | unrelated, score=0 | 局部上下文评分 |

### 7.3 目标股票相关性判定

**位置**: `news_scope.py` — `extract_target_context()`

```
输入: text, stock_name, window=80
输出: {target_relevance, target_context, reason}

逻辑:
  stock_name 不在 text 中 → unrelated
  stock_name 在 text 中 → 截取 [pos-80, pos+len(name)+80] 作为 target_context
```

## 8. 多事件冲突处理

### 8.1 冲突场景

同一条新闻可能包含多个方向相反的事件。例如：

```
"公司净利润增长200%，但因涉嫌信息披露违法被立案调查"
```

### 8.2 合并规则

```python
positive_score = sum(所有正分数)
negative_score = sum(所有负分数)
min_negative = 最负的分数

if min_negative <= -3.0 and positive_score > 0:
    # 重大负面覆盖：负分全额 + 正分20%抵消
    final = min_negative + positive_score * 0.2
    risk_flags.append("positive_event_overridden_by_major_risk")
else:
    # 普通多空：直接相加
    final = positive_score + negative_score

final = clamp(final, -5.0, 5.0)
```

### 8.3 输出结构

```python
{
    "primary_event": "regulatory",     # 主事件（按优先级）
    "score": -3.5,                      # 合并后分数
    "sentiment": "negative",            # 重大负面覆盖
    "risk_flags": ["positive_event_overridden_by_major_risk"],
    # (debug模式下可查看 all_event_scores)
}
```

## 12. 常见问题与解决方案

### Q1: 为什么"重组"不再直接判为利好？

**答案**: 旧版将"重组"作为一个固定权重（如 +2.0）的关键词，但这忽略了重组的细分状态：

| 状态 | 正确判定 |
|------|---------|
| 终止重大资产重组 | 强利空 (-3.5) |
| 获得证监会核准 | 强利好 (+3.0) |
| 筹划中（存在不确定性） | 弱利好 (+1.2, certainty=uncertain) |
| 不构成重大调整 | 中性 (0.0) |

新版先识别事件类型 restructure，再根据关键词细分判断。

### Q2: 为什么"环比扭亏"不能覆盖"同比下降"？

**答案**: 同比和环比是两个不同维度的时间对比：
- **同比**: 今年 vs 去年同期（财报主口径）
- **环比**: 本季度 vs 上季度（辅助参考）

狮头股份案例：`"净利润同比下降9.73%，环比扭亏为盈"`
- 同比降 9.73% → 基本面同比仍是下降趋势
- 环比扭亏 → 仅说明Q1比Q4有所改善

规则：**同比优先，环比只做弱修正**（+0.2）

### Q3: score=-0.6 为什么判 negative 而不是 neutral？

**答案**: 旧版使用 ±0.8 的阈值，导致小幅负面被吞掉。修正后：
- 阈值改为 **±0.2**（情感方向判定）
- 影响等级单独映射：abs(0.6)=0.6 → `weak`（弱）

这样 score=-0.6 的输出是：sentiment=negative, impact_level=weak
- 表达了"方向是负面" + "强度只是弱"的双重信息

### Q4: 多股新闻中为什么不能直接判利好？

**答案**: 多股/市场综述文本可能同时提到几十只股票，但不能把全文出现的"回暖"、"涨停"、"超预期"等利好词错误归因给某一只股票。

例如"竞价看龙头"中提到的10只股票各有涨跌，系统必须：
1. 先判断新闻范围 → multi_stock
2. 检查目标股票是否出现
3. 只提取目标附近的局部上下文做评分

### Q5: 如何新增一种事件类型？

**步骤**:
1. 在 `constants.py` 的 `EVENT_TYPES` 中添加事件名
2. 在 `constants.py` 的 `EVENT_PRIORITY` 中添加优先级
3. 在 `event_classifier.py` 的 `EVENT_KEYWORDS` 中添加关键词
4. 创建 `rules/新事件.py`，实现评分函数
5. 在 `scorer.py` 的 `score_event()` 中添加调用分支
6. 如果需要事实抽取，在 `fact_extractor.py` 中添加抽取函数
7. 在 `analyzer.py` 的 `_extract_facts()` 中添加映射
8. 编写测试用例

**无需修改**: analyzer.py 主流程、scorer.py 的 merge 逻辑、aggregator.py

### Q6: 模块和旧版 SentimentAnalyzer 的关系？

旧版 `SentimentAnalyzer`（`services/sentiment_analyzer.py`）已被完全删除。

替代方案：
- 新模块 `news_sentiment` 以库的形式被各调用方直接引用
- 情感分析在 **查询时实时计算**，不写入数据库
- 调用方通过 `analyze_news_event(item)` 获得情感结果后，自行设置到响应中

### Q7: 确定性因子和情感方向的关系？

确定性因子**只修正分数大小**，不影响情感方向：

```python
# 示例: 筹划重组
raw_score = 1.2       # 方向正面
certainty = "preliminary"
factor = 0.55
final_score = 1.2 * 0.55 = 0.66  # 分数降低，但方向仍然是 positive
```

这样 `sentiment=positive, score=0.66, certainty=preliminary` 传达了"方向偏正面，但确定性较低"的完整信息。
