# 默认竞价接力模型回测表现分析

日期: 2026-05-18  
范围: 默认竞价接力当日涨停模型 `default_auction_t0_limit_lgbm`、历史回放样本、日线模拟回测

## 1. 结论摘要

当前回测表现不好，首要原因不是 LightGBM 参数不够复杂，而是模型目标与交易目标存在偏差。

现有模型预测的是“当日非一字涨停概率”，而日线模拟盘真正关心的是“开盘买入后，在给定持仓和止盈止损规则下能否稳定赚钱”。涨停概率高不等于盈亏比好，尤其在高开幅度较大时，向上空间有限，失败后的下跌空间反而更大。

因此，调整优先级应为:

1. 建立回测诊断报告，确认亏损来自哪些概率区间、开盘涨幅区间、竞价区间和卖出原因。
2. 将训练目标从“涨停命中”扩展到“交易收益/回撤表现”。
3. 校验历史回放样本与真实选股样本是否一致。
4. 优化回测交易规则，再用规则结果反推模型标签。
5. 最后再做模型参数、特征筛选和集成模型。

历史回放样本平均每天约 7.54 只，最大 18 只，最小 1 只。样本数量本身不算大，真实选股样本更少，目前不足以单独训练稳定模型。

从涨停命中角度看，模型并非完全无效。但回测收益差说明“按涨停概率买入”不等价于“按收益期望买入”。

### 2.3 重要特征信号

当前模型特征重要性第一名是 `open_change_pct`，明显高于其他特征。回放样本分桶显示:

| 开盘涨幅区间 | 样本数 | T0涨停率 |
|---|---:|---:|
| -3% ~ 0% | 273 | 12.45% |
| 0% ~ 3% | 695 | 18.42% |
| 3% ~ 7% | 688 | 35.32% |
| >= 7% | 712 | 63.06% |

这解释了为什么模型会偏爱高开股票。但高开越多，开盘买入后的剩余涨停空间越小，失败时亏损却可能更大。这是当前模型与交易收益目标冲突的关键风险。

## 3. 主要问题拆解

### 3.1 目标错位: 涨停概率不是收益概率

当前标签 `label_t0_limit_success` 关注当日是否涨停成功。这个目标适合评估“接力强度”，但不适合作为唯一买入排序依据。

例子:

- 股票 A 开盘 +9%，涨停概率 65%，成功收益约 +1%，失败可能 -6% 到 -12%。
- 股票 B 开盘 +2%，涨停概率 45%，成功收益约 +8%，失败可能 -3% 到 -5%。

仅按涨停概率排序会偏向 A，但按期望收益可能 B 更优。

### 3.2 高开特征过强，模型可能在选风险收益比差的票

`open_change_pct` 同时代表强度和风险。它提高涨停概率，也压缩买入后的剩余空间。当前模型把它当作最强正向信号，但回测规则没有同步限制高开风险。

需要独立验证:

- `open_change_pct >= 7%` 的 Top2 收益是否显著低于涨停命中率表现。
- 高开票的平均盈利、平均亏损、盈亏比、止损率。
- 高开票在不同市场情绪下是否表现分化。

### 3.3 回放样本与实盘样本口径可能仍有偏差

历史回放样本由 `stock_auction_open` 和本地日线数据重建。默认过滤条件包括:

- 竞昨比 4%-30%
- 竞价换手率 0.5%-10%
- 收盘价小于 500
- 近 100 日涨停数不少于 3
- 封板率不低于 80%
- 开盘涨幅不低于 -3%

需要确认这些条件与当前真实选股通道完全一致，尤其是:

- 流通市值是否一致。
- 近 10 日上涨是否一致。
- 停牌、ST、北交所过滤是否一致。
- 本地日线计算的封板率是否与实盘一致。
- 竞价数据口径是否一致，包括价格、竞价量、竞昨比、竞价换手率。

### 3.4 验收指标偏模型，不偏交易

当前模型中心主要看 AUC、Top1/Top3/Top5 涨停率、概率分布。实际策略是“每天按模型概率买前 2”，因此更需要以下交易口径指标:

- Top2 总收益
- Top2 胜率
- Top2 平均单笔收益
- Top2 最大回撤
- Top2 止损触发率
- Top2 跌停顺延止损触发率
- Top2 月度收益稳定性
- Top2 分市场环境收益

如果模型 AUC 提升但 Top2 收益下降，应判定模型不适合实盘排序。

### 3.5 交易规则尚未系统寻优

当前回测规则已经支持:

- 每日买入 Top N
- 最大持仓
- 最低买入概率
- 止损
- 最大持仓天数
- 高盈利阈值
- 盈利回撤卖出
- 一字板过滤
- 跌停次日卖出

但目前还没有对这些参数做系统网格搜索。单一参数组合下亏损，不代表模型完全无效，也可能是交易规则和模型排序没有匹配。

## 4. 调整方向

### 4.1 第一优先级: 回测诊断报告

新增一个诊断脚本或接口，对指定回测 run 输出:

- 按模型概率分桶: `<50`, `50-55`, `55-60`, `60-65`, `65+`
- 按开盘涨幅分桶: `-3-0`, `0-3`, `3-7`, `7+`
- 按竞昨比分桶: `4-8`, `8-15`, `15-30`
- 按竞价换手率分桶: `0.5-1`, `1-3`, `3-5`, `5-10`
- 按封板率分桶
- 按市场炸板率/情绪分桶
- 按卖出原因聚合

每个桶输出:

- 样本数
- 买入次数
- 胜率
- 平均收益
- 中位数收益
- 最大亏损
- 止损率
- 跌停顺延率

目标: 找到“模型有效但交易亏损”的具体区间。

### 4.2 第二优先级: 交易收益标签

新增训练标签，不替换原有涨停标签，先并行:

- `label_trade_profit_success`: 按当前回测规则，最终收益 > 0。
- `label_trade_return_ge_3`: 最终收益 >= 3%。
- `label_trade_no_big_loss`: 持仓期内未触发指定大亏。
- `trade_return_pct`: 连续收益标签，用于回归或排序。

新模型可以命名为:

- `default_auction_trade_profit_lgbm`
- `default_auction_expected_return_lgbm`

排序方式从单一涨停概率改为:

```text
综合买入分 = 预期收益分 + 涨停概率分 - 风险惩罚分
```

### 4.3 第三优先级: 交易规则网格搜索

对现有模型先做规则寻优，不急着重训。

建议参数:

| 参数 | 候选值 |
|---|---|
| 最低买入概率 | 45, 50, 55, 60, 65 |
| 开盘涨幅上限 | 5, 7, 9, 无 |
| 止损 | -3, -5, -7 |
| 最大持仓天数 | 1, 2, 3 |
| 高盈利阈值 | 8, 10, 13, 15 |
| 回撤卖出 | 3, 5, 7 |

输出按最大回撤约束排序，而不是只按收益排序。例如:

1. 最大回撤不超过 15%。
2. 交易次数不少于指定数量。
3. 总收益最高。
4. 月度收益更稳定者优先。

### 4.4 第四优先级: 样本一致性审计

新增 `replay_vs_real_selection_audit`:

- 对最近真实选股日，重放同日历史回放选股。
- 比较真实选股列表与 replay 列表。
- 输出漏选、误选和原因。
- 分析每个差异股票的过滤链路。

验收目标:

- 最近真实选股日平均重合率达到 80% 以上。
- 差异必须能解释，例如竞价数据源不同、停牌数据缺失、日线未同步。

### 4.5 第五优先级: 特征增强

优先添加收益/风险相关特征:

- `up_limit_gap_pct`: 开盘价距离涨停价的空间。
- `open_change_bucket`: 开盘涨幅分层。
- `prev_limit_up_quality`: 昨日涨停质量。
- `prev_zhaban_count`: 近期炸板次数。
- `prev_volume_ratio`: 昨日成交量相对前日。
- `rise_3d_pct`, `rise_5d_pct`: 短期涨幅。
- `intraday_low_drawdown_pct`: 当日最大回撤标签。
- `market_regime`: 市场环境分类。
- `sector_strength_rank`: 板块强度排名。
- `stock_sector_position`: 个股在板块内地位。

这些特征比单纯增加模型复杂度更重要。

## 5. 推荐执行顺序

### 阶段 A: 不改模型，先定位亏损来源

1. 开发回测诊断报告。
2. 对 run 5、run 6、run 7 做同口径诊断。
3. 输出“最亏分桶”和“最赚分桶”。
4. 确认是否存在高开高概率但负收益的区间。

### 阶段 B: 不重训，先调交易规则

1. 开发回测参数网格搜索。
2. 使用当前 active 模型跑参数矩阵。
3. 选出最大回撤可接受的参数组合。
4. 固化为默认回测参数，而不是凭感觉设置。

### 阶段 C: 重建训练目标

1. 将当前回测买卖规则用于历史样本，生成交易收益标签。
2. 训练收益型模型。
3. 与当前涨停概率模型做 A/B 回测。
4. 若收益型模型 Top2 收益和回撤更优，再接入页面。

### 阶段 D: 样本一致性治理

1. 对真实选股日执行 replay 对照。
2. 修复数据源差异。
3. 将一致性报告接入自动学习前置验收。

### 阶段 E: 特征和模型迭代

1. 增加收益/风险特征。
2. 做特征分桶 lift 报告。
3. 对比 LightGBM 分类、回归、排序模型。
4. 仅激活通过交易回测验收的模型。

## 6. 需要避免的误区

### 6.1 不要只提高最低买入概率

最低概率过滤可以减少交易，但不一定提高收益。当前 active 模型最大概率只有约 71%，概率本身也未经过收益校准。需要先看不同概率桶的真实收益。

### 6.2 不要只追 AUC

AUC 衡量排序能力，但不衡量 Top2 交易收益。AUC 更高的模型可能因为更偏好高开票而收益更差。

### 6.3 不要直接删除低胜率样本

低胜率样本可能是模型学习风险边界的关键。应先通过分桶和特征贡献判断，不能简单清洗掉。

### 6.4 不要用真实选股 46 条样本下结论

真实样本目前太少，只能用来做一致性检查和方向参考，不能作为训练或最终验收的唯一依据。

## 7. 建议新增验收标准

后续模型激活前，应同时满足:

| 维度 | 建议门槛 |
|---|---|
| 样本覆盖 | replay 样本不少于 200 个交易日 |
| Top2 交易数 | 测试集不少于 60 笔 |
| Top2 总收益 | 大于 0 |
| 最大回撤 | 不超过 15%-20% |
| 止损率 | 不高于 35% |
| 跌停顺延止损率 | 越低越好，超过阈值需人工确认 |
| 月度稳定性 | 不允许只靠单月大赚 |
| replay/real 一致性 | 最近真实选股日重合率达到 80% 以上 |

## 8. 下一步建议

建议先开发两个小工具:

1. `回测诊断报告`: 针对某个 run_id 做分桶分析和卖出原因分析。
2. `参数网格回测`: 批量跑最低概率、开盘涨幅上限、止损、持仓天数、高盈利阈值和回撤阈值组合。

这两个工具能先回答“亏在哪里”和“当前模型有没有可用交易规则”。只有这个结论清楚后，再进入收益标签模型开发。
