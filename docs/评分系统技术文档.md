## 1. 系统概述

### 1.1 系统定位

评分系统 V2（Score V2）是选股通知系统的核心决策模块。它与选股策略、AI 综合简报相互独立，共同构成三层架构：

```text
+------------------+     +------------------+     +-------------------+
|  选股策略层       | --> |  评分系统层       | --> |  AI 综合简报层     |
|  9:25-9:30 选股   |     |  量化评分+风险调  |     |  结构化简报生成    |
|  硬过滤+条件筛选   |     |  制+决策建议       |     |  AI 标签+建议      |
+------------------+     +------------------+     +-------------------+
```

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 不复读选股策略 | 评分系统不能把"涨停/连板/封板率"作为核心理由 |
| AI 不参与评分 | AI 仅用于综合简报生成，不计算 final_score |
| LightGBM 预留 | 模型字段保留，当前版本不启用 |
| data_status 统一 | 所有模块必须返回明确的数据状态 |
| 异步生成 | 综合概览由后台异步 AI 生成，点开详情不实时调用 |

### 1.3 评分流程

```text
候选股列表
    ↓
Alpha 评分 (6维数据驱动)
    ↓
风险评分 (8维独立计算)
    ↓
最终评分 (风险调制融合)
    ↓
决策引擎 (操作建议+仓位)
    ↓
开盘预案 (4种开盘场景)
    ↓
数据库持久化 (stock_score_v2 + breakdown)
```

## 4. 核心评分算法

### 4.1 最终评分公式

```
# 模型不可用时（当前版本）:
raw_score = alpha_score

# 风险调制:
risk_adjustment = 1 - min(risk_score, 80) / 120
final_score = raw_score × risk_adjustment
```

### 4.2 计算示例

```text
股票A:
  alpha_score = 63.5
  risk_score = 34.0

  raw_score = 63.5
  risk_adjustment = 1 - min(34, 80) / 120 = 1 - 0.283 = 0.717
  final_score = 63.5 × 0.717 = 45.53

  score_grade = D
  risk_level = "中等偏低"
```

### 4.3 等级划分

| final_score 区间 | 等级 | 含义 |
|:----------------:|:----:|------|
| ≥ 85 | S | 高优先级 |
| 75 - 84 | A | 重点观察 |
| 65 - 74 | B | 普通观察 |
| 50 - 64 | C | 谨慎观察 |
| < 50 | D | 不建议参与 |

## 5. Alpha 评分模块

### 5.1 定位

评价股票的上涨潜力，满分 100 分。

**核心问题**：这只候选股有没有上涨潜力？

### 5.2 六维评分结构

| 维度 | 满分 | 数据来源 | 核心指标 |
|------|:----:|---------|---------|
| 交易价值 | 25 | 历史分桶统计 | 历史相似样本胜率、冲高5%概率、平均收益 |
| 预期收益 | 20 | 历史分桶统计 | 预期收益、预期回撤、盈亏比 |
| 流动性 | 20 | 实时数据 | 流通市值、竞价换手率、竞昨比 |
| 板块地位 | 15 | 行业数据 | 所属行业活跃度 |
| 事件驱动 | 10 | Anspire API | 是否有正向新闻催化 |
| 市场环境 | 10 | 静态参考 | 市场整体状况 |

### 5.3 每个维度的输出格式

```json
{
  "name": "交易价值",
  "score": 15.0,
  "max_score": 25,
  "metrics": {
    "historical_success_rate": "52%",
    "hit_5pct_rate": "28%",
    "avg_return": "+1.8%"
  },
  "reason": "历史相似样本胜率约52%，冲高5%概率约28%，平均收益约+1.8%",
  "data_status": "available"
}
```

### 5.4 禁止使用的理由

Alpha 评分不仅不能使用以下理由作为评分依据：

```text
✗ 涨停次数多
✗ 封板率高
✗ 竞价腾踊
✗ 昨日涨停
✗ 连板
✗ 短期涨幅大
```

这些数据可以作为**市场行情北京**出现在异动解读中，但**不克不及作为 Alpha 评分的核心理由**。

## 6. 风险评分模块

### 6.1 定位

评价股票的下行风险，满分 100 分，**分数越高代表风险越高**。

**核心问题**：这只票那里危险？为什么不适合追？

### 6.2 八维风险结构

| 维度 | 满分 | 指标 |
|------|:----:|------|
| 高位风险 | 20 | 10日涨幅 >45% 全额扣分 |
| 历史回撤风险 | 18 | 封板率 <70%、触板封板比 <50% |
| 炸板/失败风险 | 15 | 封板率 <60%、触板与封板差 |
| 流动性风险 | 15 | 竞价换手 <0.3%、竞昨比 <3% |
| 市场环境风险 | 15 | 整体市场状况（MVP 静态参考） |
| 事件缺失风险 | 10 | 缺少基本面催化、高开无支撑 |
| 财务/公告风险 | 12 | 财务数据接入情况 |
| 波动风险 | 5 | 10日波动率 |

### 6.3 风险等级

| risk_score 区间 | 等级 |
|:---------------:|:----:|
| 0 - 20 | 偏低 |
| 21 - 40 | 中等偏低 |
| 41 - 60 | 中等偏高 |
| 61 - 80 | 偏高 |
| 81 - 100 | 极高 |

### 6.4 每个维度的输出格式

```json
{
  "name": "高位风险",
  "score": 18,
  "max_score": 20,
  "metrics": {
    "pct_chg_10d": "73.8%",
    "pre_change_pct": "9.99%"
  },
  "reason": "近10日涨幅73.8%，短期累积风险极高；前日涨停，存在连板失败风险",
  "data_status": "available"
}
```

## 7. 最终评分与风险调制

### 7.1 风险调制原理

风险调制的核心思想是：**风险较高时，即使上涨潜力强，也会被明显降权**。

```text
risk_adjustment = 1 - min(risk_score, 80) / 120
```

- 当 risk_score = 0 时：risk_adjustment = 1.0（无调整）
- 当 risk_score = 30 时：risk_adjustment = 0.75（降低 25%）
- 当 risk_score = 60 时：risk_adjustment = 0.5（降低 50%）
- 当 risk_score = 80 时：risk_adjustment = 0.333（降低 66.7%）
- 当 risk_score ≥ 80 时：risk_adjustment = 0.333（上限）

### 7.2 风险调制曲线

```
risk_adjustment
1.0 |  ●
0.8 |    ●
0.6 |       ●
0.4 |          ●
0.2 |              ●
0.0 |____________________●
     0  20  40  60  80  100  risk_score
```

## 8. 决策引擎

### 8.1 决策规则

| 条件 | action_level | position_suggestion |
|------|-------------|-------------------|
| risk_score ≥ 80 | 剔除 | 不参与 |
| final_score ≥ 85 且 risk_score ≤ 30 | 可小仓试错 | 10%-15% 观察仓 |
| final_score ≥ 75 且 risk_score ≤ 40 | 重点观察 | 5%-10% 试错仓 |
| alpha_score ≥ 80 且 risk_score > 60 | 只看不买 | 不参与 |
| final_score ≥ 65 | 谨慎观察 | ≤5% 小仓 |
| risk_score > 60 | 高危险 | 不参与 |
| final_score < 50 | 不关注 | 不参与 |

### 8.2 决策输出格式

```json

  "take_profit_suggestion": "冲高4%以上分批止盈",
```

## 9. 开盘预案

### 9.1 定位

围绕当天 9:30 开盘后的交易场景生成预案。

**禁止表述**：明天、次日、第二天、隔日

### 9.2 四种开盘场景

| 场景名称 | 条件 | 建议行动 |
|---------|------|---------|
| 高开强承接 | 开盘 > 3% 且竞价量能充足 | 纳入重点观察，但不追高 |
| 高开低走 | 开盘 5 分钟内放量下杀 | 取消参与，不接飞刀 |
| 平开震荡 | 横盘整理等待方向 | 等待突破确认后再决定 |
| 低开走弱 | 低开且继续走低 | 不参与，等待下一交易日 |

### 9.3 输出结构

```json
{
  "general_plan": "小幅高开，观察9:30-9:35成交量变化，确认承接",
  "action_level": "重点观察",

  "opening_scenarios": [
    {"name": "高开强承接", "condition": "...", "action": "...", "probability": "medium"},
    {"name": "高开低走", "condition": "...", "action": "...", "probability": "medium"},
```

## 10. 异动解读

### 10.1 定位

异动解读回答"市场可能在炒什么"，**不回答"为什么符合选股策略"**。

### 10.2 数据来源

| 来源 | 接口 | 说明 |
|------|------|------|
| 新闻/公告 | Anspire API | 新闻标题、摘要、情感分析 |
| 行情盘口 | 选股结果 | 涨跌幅、竞价数据（仅作为行情北京） |

### 10.3 核心原因（仅允许）

```text
✓ 公告：业绩暴增、中标、重大合同
✓ 财报：营收/净利润同比增长
✓ 资本事件：高送转、分红、回购、增发
✓ 行业催化：政策、题材新闻
```

### 10.4 禁止作为核心原因

```text
✗ 涨停/大涨/大跌
✗ 连板
✗ 竞价抢筹
✗ 短期涨幅大
✗ 活跃涨停股
✗ 主力资金关注（禁止硬编）
```

以上信息只允许放入"行情北京"背景卡。

### 10.5 无明确催化时的兜底

```text
data_status: "generated_from_market_only"
summary_text: "未匹配到明确催化，仅行情异动"
```

| `available` | 数据可用 | 全部模块 |
| `partial` | 部分数据可用 | 综合概览、异动解读 |
| `missing` | 数据缺失 | 全部模块 |
| `not_integrated` | 功能未接入 | 龙虎榜、业绩排雷 |
| `source_not_configured` | 数据源未配置 | 新闻、龙虎榜 |
| `fetch_failed` | 拉取失败 | 全部模块 |
| `pending` | 后台生成中 | 综合概览 |
| `fallback_generated` | 使用兜底结果 | 综合概览 |
| `reserved` | 功能预留 | 模型预测 |
| `disabled` | 功能关闭 | 模型预测 |
| `not_on_list` | 未上榜 | 龙虎榜 |
| `generated_from_market_only` | 仅行情生成 | 异动解读 |

### 12.2 前端状态显示规则

| data_status | 前端显示文案 |
|------------|-------------|
| `not_integrated` | "XX 功能暂未接入，数据源未配置（not_integrated）" |

## 13. 异常处理机制

### 13.1 降级策略

| 异常场景 | 处理方式 | 影响范围 |
|---------|---------|---------|
| Anspire API 不可用 | 异动解读返回 no_data，新闻舆情返回空 | 仅影响新闻/异动模块 |
| Tushare 不可用 | 封板率使用缓存 | 仅影响封板率 |
| 评分 V2 计算异常 | 不影响旧评分系统，日志记录 | 评分 V2 无数据 |
| 模型文件不存在 | 自动跳过模型评分，使用纯规则评分 | 无 model_score |
| 数据库写入失败 | 不影响选股主流程，日志记录 | 评分 V2 未持久化 |

### 13.2 关键代码中的异常处理

```python
# scoring_service.py - 评分 V2 主流程
try:
    v2_service = StockScoringV2Service()
    v2_service.score_batch(...)
except Exception as e:
    logger.warning(f"评分V2计算失败（不影响主流程）: {e}")
```

### 13.3 前端空数据处理

所有字段为空时不允许页面报错，必须展示友好状态：

```javascript
// 空值安全访问模式
const score = data?.score?.final_score ?? '--'
const items = data?.news?.articles ?? []
const breakdown = Array.isArray(data?.alpha_breakdown) ? data.alpha_breakdown : []
```

## 15. 术语解释

| 术语 | 英文 | 说明 |
|------|------|------|
| Alpha 评分 | Alpha Score | 上涨潜力评分，满分 100 |
| 风险评分 | Risk Score | 下行风险评分，满分 100，越高越危险 |
| 最终评分 | Final Score | 风险调制后的综合评分 |
| 风险调制 | Risk Adjustment | 根据风险评分降低最终得分的机制 |
| 操作建议 | Action Level | 决策引擎输出的具体操作建议 |
| 仓位建议 | Position Suggestion | 建议的交易仓位比例 |
| 开盘预案 | Opening Plan | 围绕当天 9:30 开盘后的交易预案 |
| 异动解读 | Anomaly Interpretation | 基于公告/新闻/行情的异动原因分析 |
| 评分版本 | Score Version | 当前为 score_v2.0 |
| data_status | Data Status | 统一数据状态枚举 |
| LightGBM | LightGBM | 机器学习模型（当前版本预留不启用） |
| Anspire API | Anspire API | 新闻/公告/研报数据源 |
| 分桶统计 | Bucket Statistics | 按因子值分桶的历史样本表现统计 |
| 风险调制除数 | Adjustment Divisor | 120（风险调制公式中的分母） |
| 高风险阈值 | Max Risk for Adjustment | 80（超过此值的风险不再增加扣减） |
