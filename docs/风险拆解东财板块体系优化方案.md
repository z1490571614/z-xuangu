# 风险拆解东财板块体系优化方案

更新时间：2026-05-08

## 结论

风险拆解模块需要重新设计板块相关的数据源和算法，不建议只做接口名替换。现有实现把同花顺板块体系、同花顺资金流、手写行业映射、涨停原因、新闻主题、个股静态板块混在一条链路里，导致主线命中、板块强度、风险扣分之间互相干扰。

推荐方案是：涨停标签继续使用同花顺涨停池，板块归一、成分关系、板块行情、板块资金流统一迁移到东方财富数据体系。

核心链路调整为：

```text
同花顺涨停标签 / 新闻主题 / 选股行业字段
    -> 东方财富板块词典归一
    -> 东方财富成分关系确认
    -> 东方财富行情、资金、强度数据计算
    -> 风险拆解与龙头题材共用同一板块上下文
```

## 当前问题

### 1. 数据源体系不一致

当前代码中，风险拆解主要依赖：

- `ths_index`：同花顺板块词典
- `ths_member`：同花顺成分关系
- `ths_daily`：同花顺板块行情
- `moneyflow_ind_ths` / `moneyflow_cnt_ths`：同花顺行业/概念资金流
- `INDUSTRY_CODE_MAP`：手写同花顺行业映射
- `limit_list_ths`：同花顺涨停标签

问题在于：同一只股票的“涨停原因”“行业字段”“概念字段”“板块行情”可能来自不同语义体系，名字相似但代码和边界不同。最终表现就是：主线命中对了，但后续板块强度、行业风险、资金风险不稳定。

### 2. 风险拆解板块维度过窄

当前 `_calc_sector_risk()` 主要看：

- 板块涨跌幅
- 板块资金净流出
- 缓存主题归因说明

但短线风险不只来自板块下跌，还来自：

- 板块强度衰减
- 成交额放大但价格走弱
- 板块内涨停扩散不足
- 个股在板块内地位弱
- 板块资金流与涨停表现背离
- 主线板块热度退潮

所以算法层面需要从“行业风险”升级为“板块风险 + 题材承接风险”。

### 3. 静态行业映射维护成本高

`INDUSTRY_CODE_MAP` 是硬编码表，存在三个问题：

- 新板块无法及时覆盖
- 同义表达靠人工补
- 一旦数据源从同花顺切到东财，整张表都失效

应改为启动时同步东财板块词典，运行期用本地数据库做归一和模糊匹配。

### 4. 同花顺涨停标签不应废弃

涨停标签仍然有价值，因为它更接近当日打板语境。需要保留：

- `limit_list_ths` 的 `lu_desc`
- `limit_list_ths` 的 `lu_tag`
- 选股阶段已落库的 `lu_desc` / `lu_tag`

但这些标签只作为“事件文本”和“短线语义来源”，不再作为板块体系本身。标签要匹配到东财标准板块后再参与计算。

## 目标架构

### 新服务：`DcBoardService`

建议新增 `backend/services/dc_board_service.py`，不要在原 `ThsBoardService` 上继续改名堆逻辑。

职责：

- 启动时同步东财板块词典
- 选股后预热入选股票的东财板块关系
- 运行期提供本地板块归一
- 查询东财板块行情
- 查询东财板块资金流
- 计算板块强度快照

推荐方法：

```python
sync_board_index_catalog(trade_date: str) -> Dict
refresh_stock_boards(stocks: Iterable[Dict], trade_date: str) -> Dict
get_stock_boards(ts_code: str, trade_date: str, refresh_if_missing: bool = False) -> List[Dict]
normalize_board_terms(text: str, source: str, top_n: int = 5) -> List[Dict]
get_board_daily(board_code: str, trade_date: str) -> Dict
get_board_moneyflow(board_code: str, trade_date: str) -> Dict
get_board_strength(board_code: str, trade_date: str) -> Dict
```

### 数据表

建议不要继续使用 `ths_` 命名。新增通用表，保留旧表只读兼容一段时间。

```text
board_index
  board_code
  board_name
  board_type
  source              # eastmoney
  trade_date

stock_board_member
  ts_code

board_daily_snapshot
  board_code
  trade_date

board_strength_snapshot

```

如果希望小步迁移，也可以先复用现有 `ths_board_index` / `stock_ths_board_member` 表，但字段 `source` 写 `eastmoney`。中期仍建议改通用表名。

## 接口替换矩阵

| 当前用途 | 当前接口/实现 | 建议替换 | 说明 |
|---|---|---|---|
| 板块词典 | `ths_index` | `dc_index`，可辅以 `dc_concept` | 启动时同步一次，运行期只读本地库 |
| 个股板块关系 | `ths_member` | `dc_member`，可辅以 `dc_concept_cons` | 选股后只预热入选股 |
| 板块行情 | `ths_daily` | `dc_daily` | 风险拆解、题材强度共用 |
| 板块资金流 | `moneyflow_ind_ths` / `moneyflow_cnt_ths` | `moneyflow_ind_dc` | 东财板块资金流 |
| 个股资金流 | `moneyflow` | 优先 `moneyflow_dc`，保留 `moneyflow` 兜底 | 个股资金风险口径更统一 |
| 大盘资金流 | `moneyflow_hsgt` | 可新增 `moneyflow_mkt_dc` | 北向仍可保留，东财大盘资金用于市场风险 |
| 涨停标签 | `limit_list_ths` | 保留 | 只作为涨停语义文本，不作为板块体系 |
| 涨跌停/炸板统计 | `limit_list_d` / `limit_list_ths` | `limit_list_d` 为主，`limit_list_ths` 保留标签 | 市场情绪建议统一用 `limit_list_d` |
| 龙头热板块 | `limit_cpt_list` | 东财板块强度自算，`limit_cpt_list` 降为参考 | 避免热榜缺板块导致主线断链 |

## 风险拆解算法重设计

### 总体权重建议

当前权重：

```text
market 12
chip 15
news 20
capital 15
lhb 10
sector 10
technical 18
```

建议调整为：

```text
market 10        市场环境
chip 14          筹码压力
news 18          舆情与公告
capital 14       个股资金
lhb 10           龙虎榜
sector 18        板块与题材风险
technical 16     技术结构
```

理由：短线接力中，板块退潮、题材强弱、个股在板块中的地位，对风险解释力比原来的 10 分更高。

### 板块风险新算法

板块风险建议拆成 5 个子因子，最终压缩到 18 分。

| 因子 | 分值 | 数据源 | 说明 |
|---|---:|---|---|
| 板块行情走弱 | 0-5 | `dc_daily` | 当日/前日涨跌幅、连续下跌 |
| 板块资金流出 | 0-4 | `moneyflow_ind_dc` | 资金净流出、流出强度 |
| 涨停扩散不足 | 0-4 | `dc_member + limit_list_d` | 板块成分里涨停家数、占比 |
| 个股板块地位弱 | 0-3 | `dc_member + selected_stock` | 个股涨幅/连板/封单在板块内排名 |
| 题材衰减/拥挤 | 0-2 | 历史 `board_strength_snapshot` | 连续热度高后回落，或大涨后放量滞涨 |

建议规则：

```text
板块跌幅:
  pct_chg <= -3%       +5
  pct_chg <= -1.5%     +3
  pct_chg <= -0.5%     +1
  pct_chg >= 3%        -2

板块资金:
  net_amount <= -5亿   +4
  net_amount <= -2亿   +2
  net_amount >= 5亿    -2

涨停扩散:
  limit_up_count == 0                 +4
  limit_up_count <= 2 and member多     +2
  limit_up_ratio >= 5%                -2

个股地位:
  个股非板块涨幅前20%                 +2
  个股无连板且板块已有高标             +1
  个股为板块涨停高标                  -2

题材衰减:
  strength_score 连续2日下降           +2
  成交额放大且涨幅转负                +1
```

### 个股资金风险

当前使用 `moneyflow`，建议：

- 优先 `moneyflow_dc`
- 降级 `moneyflow`

评分保持 15 分以内：

```text
主力净流出 > 5000万       +10
主力净流出 > 2000万       +6
主力净流出 > 500万        +3
主力净流入 > 5000万       -2
```

如果 `moneyflow_dc` 有大单/超大单字段，应优先看超大单净额，不只看总净额。

### 市场情绪风险

当前市场情绪混用 `limit_list_ths` 与 `limit_list_d`。建议：

- 涨跌停/炸板：`limit_list_d`
- 连板高度：`limit_step`
- 大盘行情：`index_daily`
- 大盘资金：新增 `moneyflow_mkt_dc`
- 北向资金：`moneyflow_hsgt` 保留为辅助

算法不必大改，但数据口径要统一，避免涨停家数来自同花顺，炸板率来自另一个接口。

### 新闻与公告风险

当前新闻风险依赖 `integrated_news_service`，符合项目规则，应保留。

建议优化点：

- 负面新闻只计算与个股强关联的新闻
- 行业新闻只进入板块风险，不进入个股利空
- 新闻主题关系输出的 `primary_theme` 应写入板块上下文，供风险拆解读取
- 公告类关键词继续用规则，但要区分“个股公告”和“市场综述”

### 龙虎榜风险

当前复用 `lhb_service`，符合项目规则，不建议改数据源。

优化点：

- 龙虎榜席位标签仍是风险解释主力
- 强势依据中应显示纯龙虎榜高溢价席位名称，例如“高溢价席位买入：大连黄河路、相城大道”
- 风险依据中应显示纯龙虎榜砸盘/弱势席位名称，例如“砸盘席位卖出：华泰成都南一环路、东方财富拉萨”
- 高溢价席位只在风险分中抵扣风险，不把风险分显示为负数；正向价值应进入强势依据或龙虎榜 alpha
- 砸盘席位包括核按钮、量化卖出、散户集中接盘/卖出等弱势标签，展示时保留具体席位名
- 若板块风险高且龙虎榜净卖出，风险联动加分
- 若板块强但个股龙虎净卖出，提示“强板块内部分歧”

建议输出两个独立的可见字段，避免只显示抽象标签：

```json
{
  "lhb_strength_evidence": [
    {
      "type": "premium_buy",
      "label": "高溢价席位买入",
      "seats": ["大连黄河路", "相城大道"],
      "score_effect": "抵扣龙虎风险，进入强势依据"
    }
  ],
  "lhb_risk_evidence": [
    {
      "type": "dump_sell",
      "label": "砸盘席位卖出",
      "seats": ["华泰成都南一环路", "东方财富拉萨"],
      "score_effect": "增加龙虎风险"
    }
  ]
}
```

展示规则：

```text
纯高溢价买入:
  强势依据显示席位名称
  龙虎风险分最低为 0，不显示负风险

高溢价买入 + 散户/量化/核按钮风险:
  强势依据显示高溢价席位
  风险依据显示砸盘或弱势席位
  风险分按净风险计算

纯砸盘席位:
  风险依据显示席位名称
  强势依据不展示龙虎榜强势项
```

### 技术面风险

当前使用 `stk_factor_pro`，可保留。

建议减少重复惩罚：

- MACD、KDJ、RSI 同时高位时设置上限
- 技术风险在一字板、缩量封板场景下权重降低
- 技术风险应更多作为“兑现风险”而不是独立否定题材

## 风险拆解输出建议

当前 `sector_tips` 容易出现“匹配板块：xxx”这类内部说明。建议输出分为：

```json
{
  "sector_score": 8,
  "sector_tips": [
    "算力租赁板块下跌2.1%，题材承接转弱",
    "板块资金净流出3.2亿",
    "板块内涨停扩散不足，仅2家涨停"
  ],
  "sector_context": {
    "primary_board": {
      "code": "BKxxxx",
      "name": "算力租赁",
      "source": "eastmoney"
    },
    "board_pct_chg": -2.1,
    "money_net_amount": -3.2,
    "limit_up_count": 2,
    "member_count": 80,
    "strength_score": 42
  }
}
```

用户可见文案应描述市场含义，不展示“归一、匹配、来源”等内部实现词。

强势依据和风险依据建议同时接入龙虎榜席位证据：

```json
{
  "strength_evidence": [
    "高溢价席位买入：大连黄河路、相城大道",
    "机构净买入：机构专用"
  ],
  "risk_evidence": [
    "砸盘席位卖出：华泰成都南一环路",
    "量化席位卖出：华鑫上海分公司"
  ]
}
```

注意：这里展示的是席位名，不展示“归一命中”“规则命中”等内部实现词。若只有高溢价席位，风险分仍显示为 0，但强势依据必须保留该席位名称，避免用户误以为龙虎榜没有参与判断。

## 迁移步骤

### 第一阶段：新增东财板块服务

新增：

- `backend/services/dc_board_service.py`
- `backend/models/board.py` 或扩展现有 `stock_ths_board.py`
- `tests/backend/unit/test_dc_board_service.py`

实现：

- `dc_index` 启动同步
- `dc_member` 入选股预热
- `dc_daily` 行情读取
- `moneyflow_ind_dc` 板块资金读取
- 本地归一匹配

验收：

- 不在每次归因时调用 `dc_index`
- `算力租赁 / 智算租赁服务 / 算力服务` 能归一到东财标准板块
- `dc_daily` 能返回板块涨跌幅并缓存

### 第二阶段：龙头题材切换东财

替换：

- `ThemeContext` 中的 `ThsBoardService` -> `DcBoardService`
- `limit_cpt_list` 热榜降级为参考
- `get_hot_boards()` 改为东财板块强度快照

保留：

- `limit_list_ths` 涨停标签

验收：

- 中嘉博创主线能命中算力租赁
- 合力泰不再被华为/人工智能宽泛概念抢主线
- 题材命中后仍有板块强度、涨停扩散、梯队判断

### 第三阶段：风险拆解板块维度重构

替换：

- `_get_ths_industry_code()` 删除
- `_get_ths_board_candidates()` 替换为东财候选
- `_resolve_sector_board()` 改为东财板块归一 + 成分确认
- `_calc_sector_risk()` 改用 `dc_daily + moneyflow_ind_dc + board_strength_snapshot`
- `_calc_market_risk()` 中的行业板块表现改用 `dc_daily`

验收：

- 行业字段缺失时，可由涨停标签/新闻主题确认板块
- 板块行情缺失时降级，不阻断其他风险维度
- 板块风险说明不出现“匹配/归一”等内部词

### 第四阶段：资金与市场情绪东财化

替换：

- 个股资金优先 `moneyflow_dc`
- 板块资金使用 `moneyflow_ind_dc`
- 大盘资金新增 `moneyflow_mkt_dc`
- 涨跌停统计统一以 `limit_list_d` 为主

保留：

- 北向资金 `moneyflow_hsgt`
- 龙虎榜 `top_list/top_inst/hm_list`
- 涨停标签 `limit_list_ths`

### 第五阶段：清理同花顺板块旧实现

待东财链路稳定后：

- 删除 `INDUSTRY_CODE_MAP`
- 将 `ThsBoardService` 降为兼容层或删除
- 旧表只读迁移后废弃
- 文档中更新“同花顺板块”相关描述

## 测试计划

### 单元测试

- 东财词典同步只在启动/显式调用时访问 Tushare
- 运行期归一只读 DB，不访问 Tushare
- 涨停标签文本能匹配东财板块
- 新闻主题优先于涨停标签
- 东财成分关系能确认个股所属板块
- `dc_daily` 缺失时板块风险降级为 0 分
- `moneyflow_ind_dc` 缺失时不影响板块行情风险

### 回归样本

重点样本：

- 中嘉博创：涨停标签含算力租赁，应命中东财算力相关板块
- 合力泰：不应被华为概念抢主线
- 行业字段为通信设备的股票：应能匹配东财通信设备/通信服务类板块
- 新闻主题强、涨停标签弱的股票：新闻主题优先
- 无新闻、无涨停标签股票：降级到东财成分板块

### 指标评估

建议跑最近 5 个交易日入选股票：

```text
主题召回率 = 有明确新闻/涨停主题且命中板块的股票数 / 有明确主题股票数
误匹配率 = 人工抽样判定主线错误数 / 抽样股票数
板块数据覆盖率 = 有 dc_daily 的主线板块数 / 命中主线板块数
风险解释覆盖率 = sector_tips 非空股票数 / 入选股票数
```

目标：

```text
主题召回率 >= 85%
误匹配率 <= 15%
板块数据覆盖率 >= 80%
风险解释覆盖率 >= 80%
```

## 需要注意的边界

- 东财接口可能没有同花顺涨停标签的同名板块，必须允许模糊匹配和别名扩展。
- 涨停标签仍然是同花顺来源，不要把它替换掉。
- 东财板块包括概念、行业、地域，地域板块默认低优先级，避免“福建/广东”等抢主线。
- 成分关系不能作为最高优先级，只能作为确认和兜底。主线仍应优先来自新闻主题和涨停标签。
- 风险拆解不能因为板块数据缺失而整体失败，缺失维度降级为 0 分并给出“暂无板块行情数据”。

## 推荐实施顺序

建议先做第一到第三阶段，不急着删除旧同花顺代码。

第一批改动完成后，系统应达到：

- 涨停标签保留同花顺
- 主题归一和板块关系使用东财
- 风险拆解板块行情和板块资金使用东财
- 旧同花顺板块服务不再参与主流程，只作为灰度回退

这样风险最小，也最容易用真实股票验证效果。
