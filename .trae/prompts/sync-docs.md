# Prompt：同步文档 (Sync Docs)

## 任务目标
扫描代码和 `specs/` 文档，发现并修复不一致——确保文档始终反映代码真实状态。

## 必读上下文
| 文件 | 目的 |
|------|------|
| `project-status.json` | 了解当前状态 |
| `specs/` 下所有文档 | 了解文档描述的内容 |
| 相关代码文件 | 了解代码实际行为 |

## 执行步骤
1. 了解用户要同步的范围（全局 / 指定功能）
2. 扫描 specs 文档和对应代码
3. 逐一检查：
   - 需求文档 AC 与实际功能是否一致？
   - 技术方案 API 与代码是否一致？
   - 数据库结构与文档是否一致？
   - 新增文件是否更新了结构文档？
4. 列出所有不一致项，与用户确认哪些需要更新
5. 增量更新文档（就地修改 + 末尾变更日志）

## 文件修改边界
**会修改**：`specs/` 下不一致的文档
**不会修改**：代码文件（发现代码需要改时，建议走 Bug 修复或功能迭代）

## 输出格式
- 更新的文档（增量修改 + 变更日志）
- 同步报告（哪些更新了、哪些跳过、哪些遗留）

## 验证要求
- [ ] 不一致项已全部处理
- [ ] 只修改了实际有差异的部分
- [ ] 变更日志记录了改动

## 禁止行为
- 不直接改代码——文档同步只更新文档
- 不确定代码行为是否正确时，先确认再改文档
- 不做全量重写——增量为主
