# Prompt：实现任务 (Implement Task)

## 任务目标
根据已确认的需求文档、技术方案、任务规划，按 TDD 循环逐任务实现功能代码。

## 必读上下文
| 文件 | 目的 |
|------|------|
| `specs/PROJECT-CONTEXT.md` | 了解项目全局 |
| `specs/features/{功能名}.md` | 需求文档（AC 是测试依据） |
| `specs/features/{功能名}_技术方案.md` | 技术方案（实现依据） |
| `specs/features/{功能名}_任务规划.md` | 任务清单（执行依据） |
| `specs/开发规范.md` | 代码风格约定 |
| `specs/项目结构.md` | 文件归属 |

## 执行步骤
1. 读取上述全部文档，建立上下文
2. 按阶段顺序执行任务，每个任务走：
   - **RED**：根据验证标准写测试 → 运行确认失败（原因：功能未实现）
   - **GREEN**：写最小代码让测试通过 → 跑全量测试防回归
   - **REFACTOR**：在测试保护下优化（简单任务可跳过）
3. TDD 通过后根据任务性质做验收：
   - UI 变化 → Playwright 浏览器验证
   - 纯后端 → 单元测试覆盖
   - 数据库变更 → 查询验证
4. 阶段完成后：跑全量测试、对照 AC 检查、更新任务规划、生成完成报告

## 文件修改边界
**会修改**：对应模块的代码文件、测试文件、`specs/features/{功能名}_任务规划.md`
**不会修改**：需求文档、技术方案、无关模块代码

## 输出格式
- 修改后的代码和测试
- 更新后的任务规划（已完成任务标记 `[x]`）
- `docs/开发记录/{功能名}_阶段N_完成报告.md`

## 验证要求
- [ ] 每个任务走完 RED → GREEN → REFACTOR 循环
- [ ] 全量测试通过（包括已有测试）
- [ ] UI 任务通过浏览器端验收
- [ ] AC 覆盖进度已更新

## 禁止行为
- 不跳过 RED 阶段直接写实现代码
- 不修改需求文档或技术方案（如果发现不一致，先报告）
- 不在一个阶段内执行所有阶段的任务
- 不删除测试让代码通过
