# Prompt：重构模块 (Refactor Module)

## 任务目标
在测试保护下对指定模块进行受控重构——改善代码质量而不改变外部行为。

## 必读上下文
| 文件 | 目的 |
|------|------|
| `specs/PROJECT-CONTEXT.md` | 项目全局认知 |
| `specs/开发规范.md` | 代码规范标准（重构目标） |
| `specs/项目结构.md` | 文件归属 |
| 要重构的代码 + 关联测试 | 了解现状 |

## 执行步骤
1. 明确重构范围（哪些文件/模块）
2. 确认已有测试覆盖（如果测试覆盖不足，先补充测试再重构）
3. 在测试保护下逐步重构：
   - 每次改动后跑测试确认通过
   - 不改行为——只改代码结构、命名、组织方式
4. 记录重构内容（改了哪、为什么）

## 文件修改边界
**会修改**：指定模块代码
**不会修改**：外部 API、数据库 schema、其他模块代码

## 输出格式
- 重构后的代码
- 重构说明（做了什么改动、为什么）
- 确保测试通过的证明

## 验证要求
- [ ] 重构前后测试全部通过
- [ ] 外部接口没有变化
- [ ] 代码质量提升（命名更清晰、耦合更低、重复消除）

## 禁止行为
- 不在没有测试保护的情况下重构
- 不改变外部行为或 API
- 不扩大重构范围（不顺手重构不相关的模块）
- 不增加新功能
