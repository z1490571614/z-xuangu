# Prompt：生成测试 (Generate Tests)

## 任务目标
为指定模块或功能生成或补充测试用例，确保关键路径、边界条件和异常处理都有测试覆盖。

## 必读上下文
| 文件 | 目的 |
|------|------|
| `specs/PROJECT-CONTEXT.md` | 项目全局认知 |
| `specs/开发规范.md` | 测试工具、策略约定 |
| 对应的 feature 文档 | 了解 AC 和期望行为 |
| 要测试的源代码 | 了解实际实现 |

## 执行步骤
1. 确定测试范围（指定模块/功能/文件）
2. 分析需要测试的行为：
   - 正常路径（Happy Path AC 中的行为）
   - 边界条件（Edge Case AC 中的行为）
   - 异常处理（错误输入、网络失败等）
3. 逐个编写测试：
   - 每个测试验证一个行为
   - 使用项目已有测试工具和风格
   - 覆盖关键分支和边界
4. 运行测试，确认通过（或发现缺失的功能）

## 文件修改边界
**会创建/修改**：测试文件（`__tests__/` 或 `.test.ts`）
**不会修改**：源代码（除非发现测试暴露了 Bug——此时建议先走 Bug 修复流程）

## 输出格式
- 生成的测试代码
- 测试覆盖说明（哪些场景有测试、哪些不需要测试）
- 运行结果

## 验证要求
- [ ] 测试可运行且通过
- [ ] 覆盖了关键路径和主要边界
- [ ] 遵循项目已有测试风格
- [ ] 没有"为了覆盖率而写"的假测试

## 禁止行为
- 不修改源代码来"通过"测试
- 不跳过明显的边界条件
- 不生成与项目测试风格不一致的测试代码
- 如果测试发现功能缺失或 Bug，不假装没看到——告知用户
