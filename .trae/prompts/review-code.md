# Prompt：代码审查 (Review Code)

## 任务目标
对代码进行结构化审查，检查正确性、规范性、设计质量、安全性和文档一致性。

## 必读上下文
| 文件 | 目的 |
|------|------|
| `specs/PROJECT-CONTEXT.md` | 项目全局认知 |
| `specs/开发规范.md` | 代码规范标准 |
| `specs/项目结构.md` | 了解文件归属 |
| 对应的 feature 文档 | 了解需求和设计意图 |

## 执行步骤
1. 读取相关 specs 和要审查的代码
2. 从以下维度审查：
   - **正确性**：是否符合 AC 和技术方案？
   - **规范性**：命名、风格、类型使用？
   - **设计**：是否复用已有模式？有无不必要耦合？
   - **安全**：输入校验、注入风险、敏感信息？
   - **测试**：核心路径是否覆盖？测试是否可信？
   - **文档一致性**：代码是否与 specs 一致？
3. 区分"严重问题（必须修）"和"建议（可选改）"
4. 输出审查报告

## 文件修改边界
**不会修改**：任何代码文件（审查只报告，不修改）

## 输出格式
审查报告：概要 → 严重问题 → 建议 → 正面反馈 → 测试建议 → 文档一致性

## 验证要求
- [ ] 覆盖了正确性、规范性、设计、安全四个维度
- [ ] 每个问题有明确位置和修复建议
- [ ] 区分了"必须修"和"建议改"

## 禁止行为
- 不给"看起来没问题"的空洞审查
- 不把个人偏好当问题
- 不在审查中提出新功能需求
- 不审查与当前任务无关的代码
